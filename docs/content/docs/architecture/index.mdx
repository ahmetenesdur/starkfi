## Key Design Patterns

### Service Layer Isolation

Each service module is self-contained вҖ” the Fibrous service doesn't know about Vesu, and vice versa. The `batch` and `portfolio` services compose across other services.

### Tool вҶ’ Handler вҶ’ Service

MCP tools use a three-layer pattern:

1. **Tool registration** (`mcp/tools/`) вҖ” Zod schema + description
2. **Handler** (`mcp/handlers/`) вҖ” Input validation + orchestration
3. **Service** (`services/`) вҖ” Business logic + SDK calls

### Concurrent Operations

Token balance checks and portfolio aggregation use a sliding-window concurrency pattern with configurable limits (default: 10 concurrent tasks).
