---
title: Staking Commands
description: Stake, unstake, claim rewards, and browse validators
---

## validators

Lists all available validators.

```bash
starkfi validators
```

Shows validator name and staker address for the current network. Use `--json` for raw output.

## pools

Lists staking pools for a specific validator.

```bash
starkfi pools <validator>
```

Accepts a validator name (e.g., `karnot`) or staker address. Shows pool address, token, amount, and commission.

## stake

Stakes tokens with a validator.

```bash
starkfi stake <amount> -v <validator> [flags]
```

**Flags:**

| Flag              | Default | Description                      |
| ----------------- | ------- | -------------------------------- |
| `-v, --validator` | —       | Validator name (auto-finds pool) |
| `-p, --pool`      | —       | Pool contract address (direct)   |
| `-t, --token`     | `STRK`  | Token to stake                   |
| `--simulate`      | `false` | Estimate fees without executing  |

You must provide either `--validator` or `--pool`.

**Supported tokens:** STRK, WBTC, tBTC, SolvBTC, LBTC

**Examples:**

```bash
starkfi stake 100 -v karnot
starkfi stake 50 -v fibrous --simulate
starkfi stake 10 -p 0x04a3... -t STRK
```

## stake-status

Shows a consolidated staking dashboard across all validators and pools.

```bash
starkfi stake-status [validator]
```

Optionally filter by validator name. Shows staked amount, pending rewards, total value, and commission for each pool. Use `--json` for raw output.

## rewards

Claims or compounds staking rewards.

```bash
# Claim rewards (withdraw to wallet)
starkfi rewards --claim -v <validator>

# Compound rewards (restake automatically)
starkfi rewards --compound -v <validator>
```

**Flags:**

| Flag          | Description                      |
| ------------- | -------------------------------- |
| `--claim`     | Withdraw rewards to wallet       |
| `--compound`  | Claim and immediately restake    |
| `--validator` | Validator name (auto-finds pool) |
| `--pool`      | Pool address (direct)            |
| `-t, --token` | Token symbol (default: `STRK`)   |

You must provide either `--validator` or `--pool`, and either `--claim` or `--compound`.

## unstake

Begins or finalizes the unstaking process (two-step with cooldown).

```bash
# Step 1 — Create exit intent
starkfi unstake intent -v <validator> -a <amount>

# Step 2 — Finalize exit (after cooldown)
starkfi unstake exit -v <validator>
```

**Arguments:**

| Argument | Description                                               |
| -------- | --------------------------------------------------------- |
| `action` | `intent` to start exit, `exit` to complete after cooldown |

**Flags:**

| Flag              | Description                      |
| ----------------- | -------------------------------- |
| `-v, --validator` | Validator name (auto-finds pool) |
| `-p, --pool`      | Pool contract address (direct)   |
| `-a, --amount`    | Amount to unstake (for `intent`) |
| `-t, --token`     | Token symbol (default: `STRK`)   |

> **Note:** There is a cooldown period between the intent and the exit. Check your position with `stake-status` to see the exit window.
