# StarkFi Authentication Server

A lightweight, secure backend proxy for the StarkFi CLI. Bridges Privy authentication (Email OTP) to app-managed Starknet wallets, ensuring the CLI never directly handles sensitive Privy API keys.

## Architecture and Security Model

The server employs a strict security architecture designed around access minimization and key isolation:

- **App-Managed Wallet Provisioning:** Starknet wallets are dynamically created via the server environment's `APP_ID` and `APP_SECRET` credentials.
- **Trusted Execution Environment (TEE) Isolation:** Cryptographic private keys generated by Privy never leave their secure Trusted Execution Environment. Neither the StarkFi server nor the CLI client have visibility into the raw private key material.
- **JWT-Based Authorization:** All proxy signing and paymaster endpoints require a verifiable JSON Web Token (JWT). This token is issued exclusively upon successful OTP verification and carries a strict 7-day expiration policy.
- **Rate Limiting:** Authentication endpoints are limited to **5 requests/min** per IP. Signing endpoints are limited to **30 requests/min** per IP to prevent abuse.
- **DDoS Mitigation:** All authentication endpoints are protected by strict rate-limiting policies to prevent brute-force attacks.

## Local Development Setup

To initialize and run the server locally:

```bash
# Install dependencies
pnpm install

# Configure environment variables
cp .env.example .env

# Initialize development server
pnpm dev
```

## Environment Configuration Parameters

The server requires specific environment variables to function correctly. Ensure these are defined in your `.env` file before execution.

| Environment Variable | Requirement  | Description                                                                              |
| -------------------- | ------------ | ---------------------------------------------------------------------------------------- |
| `PRIVY_APP_ID`       | **Required** | The primary application identifier defined in the Privy Dashboard.                       |
| `PRIVY_APP_SECRET`   | **Required** | Secret key for app-managed wallet access (from Privy Dashboard).                         |
| `JWT_SECRET`         | **Required** | A high-entropy string (minimum 32 characters) utilized for JWT signing.                  |
| `PORT`               | Optional     | The TCP port on which the server will listen (defaults to `3001`).                       |
| `NODE_ENV`           | Optional     | Runtime environment: `development`, `production`, or `test` (defaults to `development`). |
| `ALLOWED_ORIGINS`    | Optional     | A comma-separated list of fully qualified domains authorized for CORS.                   |
| `PUBLIC_URL`         | Optional     | Public-facing URL of the server (for CORS and redirects).                                |
| `AVNU_API_KEY`       | Optional     | AVNU Paymaster API key, required to process developer-sponsored transactions.            |
| `AVNU_PAYMASTER_URL` | Optional     | AVNU Paymaster endpoint URL (defaults to `https://starknet.paymaster.avnu.fi/v1`).       |

## API Endpoint Reference

The RESTful API provides the necessary primitives for client authentication and transaction proxying.

### Authentication

| Endpoint            | Auth | Rate Limit | Description                                                       |
| ------------------- | ---- | ---------- | ----------------------------------------------------------------- |
| `POST /auth/login`  | None | 5 req/min  | Initiates the authentication flow by dispatching an Email OTP.    |
| `POST /auth/verify` | None | 5 req/min  | Validates the OTP, provisions a wallet context, and yields a JWT. |

### Transaction Signing

| Endpoint             | Auth | Rate Limit | Description                                                              |
| -------------------- | ---- | ---------- | ------------------------------------------------------------------------ |
| `POST /sign/hash`    | JWT  | 30 req/min | Proxies raw Starknet payload hashes to Privy TEE for asymmetric signing. |
| `POST /sign/message` | JWT  | 30 req/min | Proxies standard messages to Privy TEE for signing.                      |

## Middleware Stack

The following middleware is applied globally to all requests:

| Middleware        | Purpose                                                |
| ----------------- | ------------------------------------------------------ |
| `requestId()`     | Assigns a unique request ID for observability          |
| `logger()`        | Logs incoming requests to stdout                       |
| `secureHeaders()` | Sets security headers (X-Content-Type-Options, etc.)   |
| `bodyLimit()`     | Enforces 1MB max request body size                     |
| `cors()`          | CORS with configurable allowlist via `ALLOWED_ORIGINS` |
